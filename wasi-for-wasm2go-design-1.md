# WASI-for-wasm2go Design v1

## Purpose

This document is the design input for implementing a standalone, general-purpose
WASI snapshot-preview1 host for wasm2go-compiled binaries. It covers what must
be built, what exists today as the starting point, confirmed architectural
decisions, precise behavioral specifications for each implementation group, and
the test strategy needed to derive a TDD execution plan.

---

## Background and Starting Point

### Source material

`/Users/whgi/src/go-exiftool-wasm/wasi.go` (~1073 lines) is the reference
implementation. It is the wasm2go-native WASI host that backs
`github.com/lbe/go-exiftool-wasm`, the package that runs ExifTool inside a
zeroperl WASM module. Key properties:

- **wasm2go-native, not a wazero port.** Memory access is via
  `*w.module.Xmemory().Slice()` â€” a direct pointer into the generated module
  struct's linear memory slice. The implementation has no dependency on wazero,
  no `api.Module`, no host-module builder.
- **Import-interface style.** The `wasiState` struct implements all
  `X`-prefixed methods (`Xfd_read`, `Xpath_open`, etc.) that the generated
  zeroperl module consumes as its WASI imports. There is no separate dispatch
  table.
- **Covers ~33 of 40 WASI preview1 functions** needed by zeroperl.wasm, as
  enumerated in `wasi-host-requirements.md`.
- **6 functions return `ENOSYS`** (the path mutation group: mkdir, rmdir, unlink,
  link, symlink, readlink).
- **4 functions return silent success** without doing real work
  (`fd_filestat_set_size`, `fd_filestat_set_times`, `path_filestat_set_times`,
  `fd_fdstat_set_flags`).
- **Several preview1 functions are entirely absent** because zeroperl.wasm does
  not import them (`args_get/sizes_get`, `clock_res_get`, `fd_pread`, `fd_pwrite`,
  `fd_advise`, `fd_allocate`, `fd_datasync`, `fd_fdstat_set_rights`, `proc_raise`,
  `sched_yield`, `sock_*`).

### Also relevant

- `wasi-host-requirements.md` â€” the exact 30-function import surface of
  `zeroperl.wasm` derived analytically from `wasm-objdump`. Specifies which
  functions must be present, which may return `ENOSYS`, and the Asyncify
  boundary implications for `fd_read` and `call_host_function`.
- `wasi-additional-recommendations.md` â€” follow-up correctness notes including
  `clock_time_get` clock-id semantics, poll_oneoff fd subscriptions, and the
  silent-success mutator issue.
- `/Users/whgi/src/go-exiftool-wasm/io.go` â€” defines the `guestIO` interface
  (`ReadStdin`, `WriteStdout`, `WriteStderr`, `CloseStdin`, `CloseAll`) that
  will be replaced by standard `io.Reader`/`io.Writer` options in the new
  module.

## Confirmed Architectural Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Scope | Full WASI preview1 general-purpose host | Not zeroperl-specific; any wasm2go-compiled binary must be able to use it |
| Module boundary | Standalone Go module | Required for "any wasm2go binary" to be true in practice |
| Guest memory access | `func() []byte` callback | Safe across memory growth; idiomatic Go; one-line adapter from wasm2go's `*Module.Xmemory().Slice()` |
| Concurrency | Single-owner per instance, no internal locking | wasm2go Modules are not goroutine-safe; sharing a `State` requires sharing the Module, which already violates the wasm2go contract |

## Standalone Module Specification

### Module path

```
github.com/lbe/wasi-wasm2go
```

Package name: `wasihost`

### Dependencies

- Go standard library only (`crypto/rand`, `encoding/binary`, `io`, `io/fs`,
  `os`, `path`, `path/filepath`, `strings`, `sync`, `time`)
- No wazero, no cfsread, no external packages

### Public API

```go
// New creates a WASI snapshot-preview1 host state. mem is called on every
// syscall invocation to obtain the current guest linear memory slice. It is
// safe to call mem() after memory growth events; wasm2go re-slices on grow.
//
// The returned *State satisfies the X-prefixed method interfaces generated by
// wasm2go for both the wasi_snapshot_preview1 and env import modules. Pass it
// as both import arguments to the generated module's New() call:
//
//   state := wasihost.New(func() []byte { return *mod.Xmemory().Slice() }, opts...)
//   mod   := zeroperl.New(state, state)
//
// State is not safe for concurrent use. Each wasm2go Module instance must
// have its own State instance, and both must be accessed from a single
// goroutine.
func New(mem func() []byte, opts ...Option) *State

type Option func(*State)

// WithArgs sets the argv for the WASI process. Used by args_get and
// args_sizes_get. Defaults to empty (no argv).
func WithArgs(args ...string) Option

// WithEnv sets the full environment variable list as "KEY=VALUE" strings.
// Used by environ_get and environ_sizes_get. Callers are responsible for
// including any PERL5LIB or other required entries.
func WithEnv(env ...string) Option

// WithMount adds a read-only guest filesystem mount. The guest path guestPath
// (e.g. "/", "/lib", "/work") is served from root. Multiple mounts may be
// added; the longest-matching guestPath prefix wins for any given guest path.
func WithMount(guestPath string, root fs.FS) Option

// WithWritableMount adds a writable guest filesystem mount. root is the
// overlay FS used for read-through (e.g. an embedded FS layered over
// os.DirFS); hostRoot is the absolute host filesystem path used for write
// operations (os.Create, os.Remove, os.Mkdir, os.Rename, os.Readlink, etc.)
// and for the primary/fallback absolute-path resolution described below.
// If root is nil, os.DirFS(hostRoot) is used.
func WithWritableMount(guestPath string, root fs.FS, hostRoot string) Option

// WithStdin sets the reader for guest fd 0. Defaults to bytes.NewReader(nil),
// which returns (0, io.EOF) immediately. Defaulting to os.Stdin is deliberately
// avoided: this host is designed for embedded server use where the process's
// own stdin is unrelated to the guest, and silently reading from it would be
// surprising and unsafe. Callers that genuinely need to pipe stdin must pass
// an explicit reader.
func WithStdin(r io.Reader) Option

// WithStdout sets the writer for guest fd 1. Defaults to io.Discard.
// Defaulting to os.Stdout is deliberately avoided for the same reason as
// WithStdin: embedded server use makes accidental stdout leakage a
// correctness hazard. Callers must pass an explicit writer to capture output.
func WithStdout(w io.Writer) Option

// WithStderr sets the writer for guest fd 2. Defaults to io.Discard.
// Same rationale as WithStdout.
func WithStderr(w io.Writer) Option

// WithTracing enables per-syscall trace logging to os.Stdout. Off by default.
func WithTracing(enabled bool) Option

// WithOwnerAssertion enables a runtime goroutine-ownership assertion. When
// enabled, any WASI call made from a goroutine other than the first one to
// call a method panics with a descriptive message. Off by default. Enable
// via WASI_ASSERT_OWNER=1 environment variable or this option.
func WithOwnerAssertion(enabled bool) Option
```

### ExitError

```go
// ExitError is the panic value raised by proc_exit. The embedding
// application's eval boundary must recover it.
type ExitError struct{ Code int32 }
func (ExitError) Error() string { return "wasi: proc_exit" }
```

### Group B â€” ENOSYS â†’ real implementations (6 path mutation functions)

These currently return `wasiENoSys` in the reference. All follow the
`resolveDirfdPath` + `mountHostPaths` pattern already established by
`path_rename` and `path_open`.

**Shared invariant for all Group B functions:**
- If `resolveDirfdPath` returns `mount == nil`: return `ENOENT`
- If mount is not writable (`!mount.writable || mount.hostRoot == ""`): return `EROFS`
- Apply primary/fallback host path resolution via `mountHostPaths`
- Map `os` errors to WASI errno via the centralized `mapOSError` helper
  (see Internal Components); canonical mappings: not-exist â†’ `ENOENT`,
  already-exists â†’ `EEXIST`, not-empty â†’ `ENOTEMPTY`, permission â†’ `EACCES`,
  other â†’ `EIO`

#### `path_unlink_file(dirfd, pathPtr, pathLen int32) int32`

- Resolves to a host path via `resolveDirfdPath` + `mountHostPaths`
- Calls `os.Remove(hostPath)`; the target must be a file (not directory)
- If `os.Remove` fails with the primary path and a fallback exists, retries with fallback
- Returns `ENOENT` if not found, `EISDIR` if target is a directory

#### `path_create_directory(dirfd, pathPtr, pathLen int32) int32`

- Resolves to a host path via `resolveDirfdPath` + `mountHostPaths`
- Calls `os.Mkdir(hostPath, 0o755)`
- Returns `EEXIST` if already exists, `ENOENT` if parent directory does not exist

#### `path_remove_directory(dirfd, pathPtr, pathLen int32) int32`

- Resolves to a host path via `resolveDirfdPath` + `mountHostPaths`
- Calls `os.Remove(hostPath)`; target must be an empty directory
- Returns `ENOTEMPTY` if directory is not empty, `ENOENT` if not found,
  `ENOTDIR` if target is a file

#### `path_readlink(dirfd, pathPtr, pathLen, bufPtr, bufLen, nreadPtr int32) int32`

- Resolves to a host path via `resolveDirfdPath` + `mountHostPaths`
- On read-only mounts: return `ENOSYS` (embedded FS has no symlinks)
- Calls `os.Readlink(hostPath)`; writes result bytes to `mem[bufPtr:bufPtr+bufLen]`
- Writes actual bytes written (capped at `bufLen`) to `mem[nreadPtr:]` as uint32
- Returns `EINVAL` if target is not a symlink

#### `path_symlink(oldPathPtr, oldPathLen, dirfd, newPathPtr, newPathLen int32) int32`

- `oldPath` is a raw string (not a fd-relative path); `newPath` is resolved via
  `resolveDirfdPath(dirfd, newPathPtr, newPathLen)` + `mountHostPaths`
- Calls `os.Symlink(oldPath, newHostPath)`
- Returns `EROFS` if new path mount is read-only, `EEXIST` if new path exists

#### `path_link(oldDirfd, oldFlags, oldPathPtr, oldPathLen, newDirfd, newPathPtr, newPathLen int32) int32`

- Resolves both old and new paths via `resolveDirfdPath` + `mountHostPaths`
- Returns `EROFS` if either mount is read-only
- Returns `EXDEV` if old and new resolve to different mount host roots
- Calls `os.Link(oldHostPath, newHostPath)`

### Group C â€” Silent success â†’ real implementations (3 functions)

These currently return `wasiESuccess` without performing the underlying OS
operation. The reference's `wasi-additional-recommendations.md` identifies this
as a correctness issue: Perl receives a success return and assumes the operation
occurred.

The correct behavior depends on whether the fd has a real OS file backing
(`osFile` wrapper over `*os.File`) versus an embedded-FS file (`fsFileWrap`
over `fs.File`). Embedded-FS files are read-only by construction; silent success
is acceptable for them. Only `osFile`-backed fds have meaningful host operations.

#### `fd_filestat_set_size(fd int32, size int64) int32`

- If `w.fds[fd].file` is an `*osFile`: call `file.Truncate(size)`;
  return `EIO` on error
- Otherwise: return `ESUCCESS` (fs.FS files are read-only)

#### `fd_filestat_set_times(fd, atim, mtim int64, fstFlags int32) int32`

- If `w.fds[fd].file` is not an `*osFile`: return `ESUCCESS`
- Interpret `fstFlags` per WASI spec:
  - bit 0 (`ATIM`): use `atim` nanoseconds
  - bit 1 (`MTIM`): use `mtim` nanoseconds
  - bit 2 (`ATIM_NOW`): use `time.Now()` for atime
  - bit 3 (`MTIM_NOW`): use `time.Now()` for mtime
- Construct `atime` and `mtime` as `time.Time` values; call
  `os.Chtimes(file.Name(), atime, mtime)`; return `EIO` on error
- If neither ATIM nor MTIM flags are set: return `ESUCCESS` (no-op)

#### `path_filestat_set_times(dirfd, flags, pathPtr, pathLen int32, atim, mtim int64, fstFlags int32) int32`

- Resolve via `resolveDirfdPath` + `mountHostPaths`
- On read-only mounts: return `ESUCCESS` (embedded paths have no mtime to set)
- Interpret `fstFlags` same as `fd_filestat_set_times` above
- Call `os.Chtimes(hostPath, atime, mtime)`; return `EIO` on error

### Group D â€” Correctness fix (clock_time_get)

`clock_time_get` currently returns `time.Now().UnixNano()` for every clock ID.

#### `clock_time_get(clockID int32, precision int64, resultPtr int32) int32`

- `clockID == 0` (`CLOCK_REALTIME`): write `time.Now().UnixNano()` to
  `mem[resultPtr:]` as uint64 little-endian; return `ESUCCESS`
- `clockID == 1` (`CLOCK_MONOTONIC`): write nanoseconds since a fixed epoch
  anchored at `State` construction time (`time.Since(w.startTime).Nanoseconds()`);
  return `ESUCCESS`
- Any other `clockID`: return `ENOSYS`
- `State` gains a `startTime time.Time` field set in `New()`

### Group E â€” New stubs and implementations (missing from reference)

These functions are absent from the reference because zeroperl.wasm does not
import them. A general-purpose WASI host must provide all preview1 functions.

#### `args_get(argv, argvBuf int32) int32`

- Writes `w.args []string` into memory: pointer array at `argv`, null-terminated
  strings packed at `argvBuf`. Same layout as `environ_get`.
- Returns `ESUCCESS`

#### `args_sizes_get(argc, argvBufSize int32) int32`

- Writes `len(w.args)` as uint32 to `mem[argc:]`
- Writes total buffer size (sum of `len(arg)+1` for each arg) as uint32 to
  `mem[argvBufSize:]`
- Returns `ESUCCESS`

#### `clock_res_get(clockID int32, resultPtr int32) int32`

- `clockID == 0` or `clockID == 1`: write `1` (nanosecond resolution) as uint64
  to `mem[resultPtr:]`; return `ESUCCESS`
- Other IDs: return `ENOSYS`

#### `fd_pread(fd, iovs, iovsLen int32, offset int64, nreadPtr int32) int32`

- Positioned read: reads `iovsLen` iov buffers from `fd` starting at `offset`,
  without updating `entry.offset`
- Like `fd_read` but uses `ReadAt(buf, offset+accumulated)` rather than
  advancing `entry.offset`
- For `fs.FS` files that do not support `ReadAt`: seek to `offset`, read, then
  seek back to original position
- fd 0 (stdin): return `EINVAL` (positioned reads on stdin are not defined)
- Returns `EBADF` for invalid fds, `EIO` on read errors

#### `fd_pwrite(fd, iovs, iovsLen int32, offset int64, nwrittenPtr int32) int32`

- Positioned write: writes iov buffers to `fd` at `offset`, without updating
  `entry.offset`
- Uses `WriteAt(buf, offset+accumulated)` on `*osFile` backing
- fd 1 and fd 2: return `EINVAL` (positioned writes on stdout/stderr not defined)
- Returns `EBADF` for invalid fds, `EINVAL` if file does not support positioned
  writes, `EIO` on write errors

#### `fd_advise(fd int32, offset, len int64, advice int32) int32`

- Advisory hint (POSIX `posix_fadvise`); no portable Go equivalent
- Return `ESUCCESS`; document as intentional no-op

#### `fd_allocate(fd int32, offset, len int64) int32`

- Reserve disk space (`fallocate`); advisory, not correctness-critical
- Return `ESUCCESS`; document as intentional no-op

#### `fd_datasync(fd int32) int32`

- Data-only sync (metadata not required)
- If `w.fds[fd].file` is `*osFile`: call `file.Sync()`; return `EIO` on error
- Otherwise: return `ESUCCESS`

#### `fd_fdstat_set_rights(fd int32, rightsBase, rightsInheriting int64) int32`

- Capability rights narrowing
- Return `ESUCCESS`; document: rights are not enforced in this host

#### `proc_raise(signal int32) int32`

- Raise a signal inside the guest; no meaningful mapping in WASM
- Return `ENOSYS`

#### `sched_yield() int32`

- Yield the scheduler
- Call `runtime.Gosched()`; return `ESUCCESS`

#### `sock_accept(fd, flags, resultFd int32) int32`
#### `sock_recv(fd, riDataPtr, riDataLen int32, riFlags int32, roDataLen, roFlags int32) int32`
#### `sock_send(fd, siDataPtr, siDataLen int32, siFlags int32, soDataLen int32) int32`
#### `sock_shutdown(fd, how int32) int32`

- Sockets are not supported in this WASI host
- All four return `ENOSYS`

## Migration: go-exiftool-wasm as Consumer

After the standalone module exists, `go-exiftool-wasm` becomes a thin consumer.
The migration is a pure mechanical wiring change with no behavioral difference:

```go
// Before (internal wasi.go with *wasm2go.Module coupling):
ws := &wasiState{}
mod := wasm2go.New(ws, ws)
initWASIState(ws, mod, gio, mounts)

// After (standalone module; mem callback captures mod):
mod := wasm2go.New(nil, nil) // placeholder
state := wasihost.New(
    func() []byte { return *mod.Xmemory().Slice() },
    wasihost.WithEnv("PERL5LIB=/lib/"+perl5Lib+":/lib/"+perl5Lib+"/wasm32-wasi"),
    wasihost.WithWritableMount("/", rootFS, cwd),
    wasihost.WithMount("/dev", devNullFS{}),
    wasihost.WithStdin(stdin),
    wasihost.WithStdout(stdout),
    wasihost.WithStderr(stderr),
    wasihost.WithTracing(os.Getenv("EXIFTOOL_WASI_TRACE") == "1"),
    wasihost.WithOwnerAssertion(os.Getenv("EXIFTOOL_WASI_ASSERT_OWNER") == "1"),
)
mod = wasm2go.New(state, state)
```

The `guestIO` interface in `go-exiftool-wasm/io.go` is replaced by the standard
`io.Reader`/`io.Writer` options. `CloseStdin`/`CloseAll` lifecycle methods stay
in `go-exiftool-wasm` at the server-coordination layer where they belong â€”
they are not WASI host concerns.

## Notes on Retained No-ops

The following behaviors are intentionally not upgraded and should be documented
in the module:

| Function | Behavior | Rationale |
|---|---|---|
| `fd_sync` | Always `ESUCCESS` | Embedded `fs.FS` has no sync semantics; `osFile` fds could call `Sync()` but ExifTool never calls this in practice |
| `fd_fdstat_set_flags` | Always `ESUCCESS` | Non-blocking I/O on guest fds has no meaning in a synchronous wasm2go host |
| `fd_fdstat_set_rights` | Always `ESUCCESS` | Rights are advisory; this host does not enforce capability model |
| `fd_advise` | Always `ESUCCESS` | No portable Go equivalent for `posix_fadvise` |
| `fd_allocate` | Always `ESUCCESS` | Disk-space reservation is advisory |
| `poll_oneoff` clock-only | Handles clock subscriptions (sleep); validates fd existence for fd subscriptions but does not model actual readiness | A real select/poll model would require async I/O infrastructure beyond the synchronous host's scope; document clearly |

## Test Strategy

This section defines the testing approach suitable for use as tdd-plan input.

### Level 1: Unit tests (no wasm required)

Each WASI function is testable in isolation against a fake `func() []byte`
backed by a plain `[]byte` buffer. The `State` is constructed with
`wasihost.New(func() []byte { return buf }, opts...)`.

The fake memory buffer for all Level 1 tests is `make([]byte, 65536)` â€” large
enough to hold any struct written by any WASI syscall without boundary checks
interfering with the behavior under test. Tests assert exact byte values at
known offsets in this buffer after each call.

**Memory layout correctness:**
- `fdstat` struct: 8 bytes fdtype+flags, then 8 bytes rights_base, 8 bytes
  rights_inheriting (24 bytes total at `ptr`)
- `filestat` struct: 64 bytes â€” dev(8), ino(8), filetype(8), nlink(8), size(8),
  atim(8), mtim(8), ctim(8) â€” written at `ptr`
- iovec layout: each iov is 8 bytes â€” buf_ptr(4) at offset 0, buf_len(4) at
  offset 4
- All multi-byte integers are little-endian

**Behaviors to test per group:**

*Environment (A):*
- `environ_sizes_get`: count and size match configured env slice
- `environ_get`: pointer array at `envPtr`, null-terminated strings at `envBufPtr`,
  each pointer points to correct string start

*Args (E):*
- `args_sizes_get`: count 0 with no args; count N with N args
- `args_get`: same layout invariants as `environ_get`

*Clocks (D, E):*
- `clock_time_get` with clockID 0: returns nanosecond value > 0
- `clock_time_get` with clockID 1: returns value that increases between calls
- `clock_time_get` with clockID 99: returns `ENOSYS`
- `clock_res_get` with clockID 0/1: returns 1 (nanosecond)
- `clock_res_get` with clockID 99: returns `ENOSYS`

*Random (A):*
- `random_get`: buffer is non-zero after call (probabilistic; use 32 bytes)

*proc_exit (A):*
- Panics with `ExitError{code}`; recovered panic has correct code

*Preopen enumeration (A):*
- `fd_prestat_get` at fd 3 with one mount: type byte 0, len matches path
- `fd_prestat_get` past all preopens: `EBADF`
- `fd_prestat_dir_name`: bytes at pathPtr match mount path

*fd_fdstat_get (A):*

  `argvBuf+0` ("a\0"), pointer at argv+4 points to `argvBuf+2` ("bc\0")

### Level 2: OS-integration tests (direct `*State` calls, no wasm binary)

**No wasm test fixture is used at this level.** This is a deliberate design
decision based on confirmed research into both candidate sources:

- **wazero's WASI tests** use `proxy.NewModuleBinary` â€” a wasm proxy module
  generated at runtime by wazero's own API â€” and invoke WASI functions via
  `api.Module.ExportedFunction(name).Call(ctx, params...)`. This is wazero's
  dynamic runtime API; wasm2go has no equivalent. None of wazero's C, Go, or
  Zig source fixtures export individual test wrapper functions; all are
  command-style programs dispatching on `os.Args[1]`. Not reusable.
- **wasm2go's testdata** is pure-compute WAT files (fib, primes, loops, etc.)
  with no WASI imports. Not relevant.

**Why no fixture is needed:** Testing a WASI *host* implementation requires
verifying that the host's `*State` methods produce correct OS effects and
correct memory writes. Both can be exercised by calling `*State` methods
directly from Go tests â€” no guest wasm binary is involved. The "integration"
in "OS-integration" refers to real filesystem operations against
`t.TempDir()`, not to the presence of a wasm guest.

**Test harness pattern:** Each Level 2 test constructs a `*State` with a
writable mount pointing at `t.TempDir()` and a synthetic
`make([]byte, 65536)` memory buffer. The test populates the buffer with input
data (path strings, iovecs, etc.), calls the `*State` method directly, then
asserts on both the return value and host-side effects (`os.Stat`,
`os.ReadFile`, `os.Readlink`, etc.).

```go
// Canonical Level 2 test harness pattern:
buf := make([]byte, 65536)
tmpDir := t.TempDir()
state := wasihost.New(
    func() []byte { return buf },
    wasihost.WithWritableMount("/", os.DirFS(tmpDir), tmpDir),
)
// Allocate a preopen dir fd by calling _initialize equivalent:
// fd 3 is the preopen for "/"
const dirfd = 3

// Write a path string into memory at offset 1024:
path := "newdir"
copy(buf[1024:], path)

// Call the WASI method directly:
errno := state.Xpath_create_directory(dirfd, 1024, int32(len(path)))

// Assert return value and host-side effect:
if errono != 0 { t.Fatalf("expected ESUCCESS, got %d", errno) }
if _, err := os.Stat(filepath.Join(tmpDir, "newdir")); err != nil {
    t.Fatalf("directory not created: %v", err)
}
```

**Iovec construction for positioned I/O tests:** `fd_pread` and `fd_pwrite`
require an iovec in the memory buffer. An iovec is 8 bytes: `buf_ptr` (4
bytes LE) at offset 0, `buf_len` (4 bytes LE) at offset 4. The Go test
writes this structure directly:

```go
// Place data buffer at offset 2048, iovec struct at offset 1024:
binary.LittleEndian.PutUint32(buf[1024:], 2048) // buf_ptr
binary.LittleEndian.PutUint32(buf[1028:], 4)    // buf_len = 4 bytes
// nread result pointer at offset 512:
errno := state.Xfd_pread(fd, 1024, 1, int64(offset), 512)
nread := binary.LittleEndian.Uint32(buf[512:])
```

**Preopen fd setup for path operation tests:** Level 2 tests that exercise
path operations (Group B, Group C path variant) need a preopen fd at index 3.
This is established by the `State` constructor when `WithWritableMount` is
called â€” the mount at index 0 occupies fd 3. Tests use `const dirfd = 3`.

**Behaviors to test per group:**

*Group B â€” path mutation:*
- `path_create_directory`: write `"newdir"` into memory; call with `dirfd=3`;
  assert `os.Stat(filepath.Join(tmpDir, "newdir"))` succeeds and is a
  directory; second call with same path returns `EEXIST`
- `path_remove_directory`: pre-create `tmpDir/emptydir`; call with that path;
  assert `os.Stat` returns `ErrNotExist`; call with a non-empty dir returns
  `ENOTEMPTY`; call with a file path returns `ENOTDIR`
- `path_unlink_file`: pre-create `tmpDir/file.txt`; call; assert file absent;
  call again returns `ENOENT`; call with a directory path returns `EISDIR`
- `path_readlink`: call `os.Symlink("target", filepath.Join(tmpDir, "link"))`
  on the host before the test; write `"link"` into memory; call
  `Xpath_readlink`; read result bytes from memory; assert they equal `"target"`;  assert `nread` equals `len("target")`
- `path_symlink`: write old path `"../target"` and new path `"mylink"` into
  memory; call `Xpath_symlink`; assert `os.Readlink(filepath.Join(tmpDir,
  "mylink"))` returns `"../target"`
- `path_link`: pre-create `tmpDir/src.txt`; write `"src.txt"` and `"dst.txt"`
  into memory; call `Xpath_link`; assert `os.SameFile` on both host paths
- All six return `EROFS` when `dirfd` resolves to a read-only mount
  (test using `WithMount` without `WithWritableMount`)

*Group C â€” real mutators:*
- `fd_filestat_set_size` on `*osFile`: pre-create a 10-byte file; call
  `Xpath_open` to get a writable fd; call `Xfd_filestat_set_size(fd, 5)`;
  assert `os.Stat` shows size 5; assert `os.ReadFile` returns exactly 5 bytes
- `fd_filestat_set_size` on `fsFileWrap`: open a file from a read-only
  `fs.FS` mount; call `Xfd_filestat_set_size`; assert returns `ESUCCESS`
  without any host-side change (no truncation)
- `fd_filestat_set_times` with `fstFlags=1` (ATIM) and `fstFlags=2` (MTIM):
  pre-create a file; construct known `atim`/`mtim` nanosecond values; call;
  assert `os.Stat(hostPath).ModTime()` matches `mtim` within 2 seconds
- `path_filestat_set_times`: same as above but via path variant; write path
  string into memory rather than opening an fd
- `fd_filestat_set_times` with `fstFlags=0`: assert no `os.Chtimes` call and
  returns `ESUCCESS` (nop when no flags are set)

*Group D â€” clock semantics:*
- `Xclock_time_get(0, 0, resultPtr)`: read uint64 from `buf[resultPtr:]`;
  assert value > `1577836800000000000` (2020-01-01 in nanoseconds)
- Call twice with `time.Sleep(time.Millisecond)` between; assert second
  value â‰Ą first (wall-clock does not go backwards)
- `Xclock_time_get(1, 0, resultPtr)` called twice without sleep: assert
  second value â‰Ą first (monotonic never decreases)
- `Xclock_time_get(1, 0, ...)` first call vs second call after 10ms sleep:
  assert difference â‰Ą `10 * time.Millisecond` in nanoseconds
- `Xclock_time_get(99, 0, resultPtr)`: assert returns `ENOSYS` (52)

*Group E â€” positioned I/O and new stubs:*
- `fd_pread`: pre-create `tmpDir/data.bin` containing `"ABCDEFGHIJ"` (10
  bytes); call `Xpath_open` to get fd; construct 1-iovec at `[1024:1032]`
  pointing to `buf[2048:2052]`; call `Xfd_pread(fd, 1024, 1, 4, 512)`;
  assert `buf[2048:2052]` == `"EFGH"`; assert `uint32(buf[512:])` == 4;
  assert subsequent `Xfd_tell` returns 0 (entry.offset not updated)
- `fd_pread` on fd 0: assert returns `EINVAL` (28)
- `fd_pwrite`: pre-create `tmpDir/out.bin` as empty; call `Xpath_open`
  writable; place `"XYZ"` at `buf[2048:]`; construct iovec pointing there;
  call `Xfd_pwrite(fd, 1024, 1, 3, 512)`; assert `os.ReadFile` returns
  `"\x00\x00\x00XYZ"`; assert `Xfd_tell` returns 0 (entry.offset unchanged)
- `fd_pwrite` on fd 1: assert returns `EINVAL` (28)
- `sched_yield`: assert `Xsched_yield()` returns 0 (`ESUCCESS`)
- `fd_datasync` on `*osFile`-backed fd: call; assert returns `ESUCCESS`
  (internally calls `Sync()`; no error expected on a writable temp file)
- `fd_datasync` on `fsFileWrap`-backed fd: call; assert returns `ESUCCESS`
  (no-op for read-only FS files)
- `args_get` / `args_sizes_get`: construct `*State` with
  `WithArgs("prog", "--flag", "val")`; call `Xargs_sizes_get(0, 4)` and
  assert count=3, buf_size=`len("prog\0--flag\0val\0")`=16; call
  `Xargs_get(8, 64)` and assert pointer array at `buf[8:]` and strings
  at `buf[64:]` match `"prog\0"`, `"--flag\0"`, `"val\0"` in order
