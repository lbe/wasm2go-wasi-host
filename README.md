A specialized **WASI snapshot-preview1** host for Go, designed to run code transpiled by [wasm2go](https://github.com/ncruces/wasm2go).

The primary goal of this project is to provide a reference implementation to confirm compliance with the [WASI preview 1](https://github.com/WebAssembly/WASI/blob/wasi-0.1/preview1/docs.md) specification when using `wasm2go`. It implements all 40 WASI preview1 functions plus the `env.call_host_function` stub required by certain modules.

This repository also includes `wasm2go-run`, a specialized runner for the [WebAssembly/wasi-testsuite](https://github.com/WebAssembly/wasi-testsuite).

While it can be used as a general-purpose host for `wasm2go` modules, it is intentionally lightweight and standard-library-only. For production-grade performance, advanced sandboxing, or more complex host integrations, more mature runtimes like `wasmtime` or `wazero` are recommended.

## Features

- **Full WASI Preview1 Support**: Implementation of all 40 system calls.
- **wasm2go-Native**: Direct memory access via guest memory callbacks—no overhead from external runtime APIs.
- **Capability-oriented Virtual Filesystem**: Support for multiple mounts, including read-only `fs.FS` (embedded files) and writable host directory mounts.
- **Standard Library Only**: No external dependencies.
- **CLI Runner**: This repository includes [`wasm2go-run`](./cmd/wasm2go-run/README.md), a tool that transpiles, compiles, and executes WASM files in one step, primarily for testing.

## Installation

```bash
go get github.com/lbe/wasm2go-wasi-host
```

## Usage

### Using as a Library

To use the host with a `wasm2go`-generated module:

```go
import (
    "os"
    "github.com/lbe/wasm2go-wasi-host"
    "github.com/your/generated/package" // module generated by wasm2go
)

func main() {
    // 1. Initialize the WASI host state.
    // The memory callback is re-invoked on every syscall to handle memory growth.
    var mod *generated.Module
    state := wasihost.New(
        func() []byte { return *mod.Xmemory().Slice() },
        wasihost.WithArgs("app-name", "arg1"),
        wasihost.WithEnv("PATH=/bin"),
        wasihost.WithWritableMount("/", "./host-data", os.DirFS("./host-data")),
        wasihost.WithStdout(os.Stdout),
        wasihost.WithStderr(os.Stderr),
    )

    // 2. Instantiate the generated module with the host state as the import resolver.
    mod = generated.New(state, state)

    // 3. Run the guest's start function (usually _start).
    defer func() {
        if r := recover(); r != nil {
            if e, ok := r.(wasihost.ExitError); ok {
                os.Exit(int(e.Code))
            }
            panic(r)
        }
    }()
    mod.X_start()
}
```

## File System Support

`wasm2go-wasi-host` provides a flexible mounting system:

- **`WithMount(guestPath, root)`**: Mounts a read-only `fs.FS` at the specified guest path.
- **`WithWritableMount(guestPath, hostRoot, root)`**: Mounts a writable host directory. It uses `root` (optional `fs.FS`) for read-through lookups and `hostRoot` for all write operations.

## Concurrency and Safety

- **Single-owner**: The `State` is not safe for concurrent use. Each `wasm2go` Module instance must have its own `State`, and both must be used from a single goroutine.
- **Goroutine Assertion**: You can enable `WithOwnerAssertion()` to panic if the host is accessed from multiple goroutines, helping catch concurrency bugs early.
